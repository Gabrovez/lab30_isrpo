## Обязательная таблица атрибутов JSON

| Атрибут / настройка             | Что делает                          | Пример                     |
|----------------------------------|-------------------------------------|----------------------------|
| **[JsonPropertyName("name")]**     | Задаёт имя поля в JSON              | "name" вместо "Name"   |
| **[JsonIgnore]**                   | Поле не попадает в JSON             | Пароли, внутренние поля    |

## Главные выводы

### 1. Что такое сериализация и десериализация

Сериализация — это процесс преобразования объекта C# в текстовый формат (JSON), который можно передать по сети или сохранить. Десериализация — обратный процесс: из JSON-строки создаётся объект C#. Например, объект `Hero { Name = "Thor" }` превращается в строку `{"name":"Thor"}`, а затем обратно в объект.

### 2. Зачем нужен `[JsonIgnore]` — приведите реальный пример

Атрибут `[JsonIgnore]` исключает поле из сериализации. Это нужно для скрытия служебных или чувствительных данных. Например, если у героя есть поле `InternalNotes` с заметками разработчика или паролями, оно не должно попадать в ответ клиенту. Без этого атрибута такие данные могли бы утечь через API.

### 3. Почему enum лучше сериализовывать строкой, а не числом

По умолчанию enum сериализуется как число (например, `Universe.Marvel → 0`). Это неудобно для клиента: нужно знать соответствие чисел и значений. При использовании `JsonStringEnumConverter` enum становится строкой (`"Marvel"`), что делает ответ понятным без дополнительной документации. Также это устойчивее к изменениям порядка значений в enum.

### 4. Что изменилось в ответе после настройки CamelCase

До настройки поля в JSON имели имена с большой буквы: `"Id"`, `"RealName"`, `"PowerLevel"`. После включения `PropertyNamingPolicy.CamelCase` они стали соответствовать соглашению JSON: `"id"`, `"realName"`, `"powerLevel"`. Это стандарт для JavaScript-клиентов и улучшает совместимость с фронтендом.
